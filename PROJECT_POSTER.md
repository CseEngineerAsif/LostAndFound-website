# LOST2FOUND
**Find it faster. Return it easier.**

---

**Problem**  
Campus lost-and-found is scattered across notice boards and social posts, making recovery slow and unreliable.

**Solution**  
A centralized web portal to report, search, and match lost and found items with photos and filters.

---

**Key Features**  
- User registration and login  
- Lost and found item reporting  
- Optional photo uploads  
- Search and filter by category, location, and type  
- Status tracking dashboard  

**Technology Stack**  
- Node.js + Express  
- EJS templates + CSS  
- `lowdb` JSON datastore  
- Multer for file uploads  

---

**Workflow**  
1. Register or log in  
2. Report lost or found items  
3. Search listings and filter results  
4. Match items and update status  

---

**Benefits**  
- Faster item recovery  
- Better visibility of reports  
- Reduced manual coordination  
- Lightweight and easy to deploy  

---

**Future Enhancements**  
- Email/SMS notifications  
- Admin moderation  
- Production database (MySQL/Postgres)
